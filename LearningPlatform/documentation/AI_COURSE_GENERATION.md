# AI Course Generation (Experimental)

The Admin AI Agent can generate a full course draft and then create the final course content in the CMS.

> Status: **Experimental**  
> Quality, structure, and output consistency can vary by model, prompt quality, and topic complexity.

## Where to use it

- Open Admin panel and go to `/admin/ai-agent`.
- Fill in course discovery inputs (topic, level, current knowledge, learning goal type, depth, time budget, model, optional focus areas, optional flashcards).
- Generate a draft, review it, refine it with follow-up prompts, then accept it.

## How it works

1. **Draft phase**
   - UI calls `POST /api/admin/ai-agent/draft`.
   - The backend generates a structured draft (subject, course, modules, lessons, tags).
   - You can iterate on the draft with additional prompt messages.

2. **Accept and generate phase**
   - UI calls `POST /api/admin/ai-agent/accept`.
   - A background run starts and returns a `runId`.
   - UI polls `GET /api/admin/ai-agent/progress/[runId]` for timeline and progress.
   - The backend creates subjects, course, modules, lessons, tasks, and optional flashcards.

3. **Provider/model selection**
   - Anthropic models (e.g. `claude-haiku-4-5`, `claude-sonnet-4-5`) use `ANTHROPIC_API_KEY`.
   - OpenAI models (e.g. `gpt-4.1`) use `OPENAI_API_KEY`.

## Access and runtime behavior

- All endpoints in this flow require admin permissions.
- Progress runs are stored in-memory (not persisted in DB).
- In development, server reloads can clear in-memory runs; in that case `GET /progress/[runId]` can return `Run not found` and you should start again.

## What gets created in CMS

- On accept, backend creates:
  - subject (if missing)
  - course
  - modules
  - lessons with theory blocks
  - tasks
  - optional flashcard deck and cards
- Generated content is created as draft/unpublished (`isPublished: false`) by default.
- If module JSON generation fails validation after retries, backend uses fallback module content so the run can continue.

## Environment requirements

Set at least one provider key in your runtime environment (for local Docker this is usually `.env`):

- `ANTHROPIC_API_KEY` for Claude models
- `OPENAI_API_KEY` for GPT models

If the selected provider key is missing or has no usable balance, generation requests will fail.

## Cost note

Cost varies by selected model and by how much content is generated.

In my test, using **Claude Haiku**, generating one simple course cost about **$0.70 USD**.

Use this as a rough reference, not a fixed price.



## Current testing notes

- So far, testing was done only with **Claude Haiku**.
- From a technical perspective, the generation flow worked without major issues.
- Educational quality was lower than expected in some outputs (more generic/surface-level text).
- This may be related to model choice; higher-capability models can improve depth and instructional quality.
- Current UI model options are:
  - `claude-sonnet-4-5`
  - `claude-haiku-4-5`
  - `gpt-4.1`
